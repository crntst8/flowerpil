# R2 Image Storage

## Purpose

Stores all uploaded images in Cloudflare R2 object storage. Image URLs in the database point to R2 CDN endpoints instead of local filesystem paths.

## How It Works

Images are processed with Sharp (resize, compress) then uploaded to R2 via AWS S3-compatible API. The upload returns a public R2 URL which is stored in the database. Legacy local file paths (`/uploads/...`) are supported for backward compatibility during migration.

### Components

**R2 Client** (`server/utils/r2Storage.js`)
- Initializes S3Client with R2 endpoint and credentials
- Exports functions: `uploadToR2()`, `deleteFromR2()`, `extractR2Key()`, `fileExistsInR2()`

**Upload API** (`server/api/uploads.js`)
- Processes images with Sharp (creates original, large, medium, small variants)
- Uploads each variant to R2 with path: `{subdirectory}/{uuid}_{size}.{format}`
- Returns R2 URLs in response

**Bio Profiles** (`server/api/bio-profiles.js`)
- Function: `processAndSaveBioImage()`
- Uploads banner (1200x400) and content images (800x600) to `bio-pages/` subdirectory

**Curators** (`server/api/curators.js`)
- Function: `processAndSaveImage()`
- Uploads profile images (200x200) to `curators/` subdirectory

**Track Artwork** (`server/api/artwork.js`)
- Function: `processAndSaveArtwork()`
- Uploads track artwork (150x150, 300x300, 600x600, 800x800) to `tracks/` subdirectory
- Used by manual artwork upload/download endpoints

**Playlist Import Service** (`server/services/playlistImportService.js`)
- Function: `processSpotifyArtwork()`
- Uploads track artwork from DSP imports (800x800, 400x400, 200x200) to `tracks/` subdirectory
- Used when importing playlists with `artwork: true` parameter
- Scheduled imports use `artwork: false` to skip downloads and use external URLs instead

**File Cleanup** (`server/utils/fileCleanup.js`)
- Function: `deleteImageFiles()` detects R2 URLs vs local paths
- Deletes from R2 if URL contains `.r2.cloudflarestorage.com`, `.r2.dev`, or custom domain
- Falls back to local filesystem deletion for legacy paths

### Data Flow

1. Client uploads image via POST to `/api/v1/uploads/image?type={type}`
2. Multer stores file in memory buffer
3. Sharp processes buffer into multiple size variants
4. Each variant uploaded to R2 with key: `{type}/{uuid}_{size}.{ext}`
5. R2 returns public URL: `https://images.flowerpil.io/{key}` (custom domain)
6. API response includes URLs for all variants
7. Client stores primary URL in database

## API/Interface

### Upload Image

**Endpoint:** `POST /api/v1/uploads/image`

**Query Parameters:**
- `type` (optional): Upload subdirectory - `playlists`, `curators`, `bio-pages`, `search-editorials`, `playlist-action-icons`, `general`

**Request:**
- Content-Type: `multipart/form-data`
- Field: `image` (file, max 10MB)
- Allowed types: `image/jpeg`, `image/png`, `image/webp`, `image/svg+xml`

**Response:**
```json
{
  "success": true,
  "data": {
    "original_name": "photo.jpg",
    "mime_type": "image/jpeg",
    "images": {
      "original": {
        "filename": "abc123.jpg",
        "url": "https://pub-xxx.r2.dev/playlists/abc123.jpg",
        "width": 1200,
        "height": 1200,
        "size": 150000,
        "format": "jpeg"
      },
      "large": { ... },
      "medium": { ... },
      "small": { ... }
    },
    "primary_url": "https://pub-xxx.r2.dev/playlists/abc123_large.jpg",
    "created_at": "2025-11-03T12:00:00.000Z"
  }
}
```

### R2 Storage Functions

**`uploadToR2(buffer, key, contentType)`**

Uploads file buffer to R2 bucket.

Parameters:
- `buffer` (Buffer): File data
- `key` (string): Path in bucket (e.g., `playlists/abc123_large.jpg`)
- `contentType` (string): MIME type (e.g., `image/jpeg`)

Returns: `string` - Public R2 URL

**`deleteFromR2(key)`**

Deletes file from R2 bucket.

Parameters:
- `key` (string): File path in bucket

Returns: `boolean` - Success status

**`extractR2Key(url)`**

Extracts R2 key from URL or legacy path. Supports:
- Custom domain URLs (e.g., `https://images.flowerpil.io/...`)
- R2.dev URLs (e.g., `https://pub-xxx.r2.dev/...`)
- R2 Cloudflare Storage URLs (e.g., `https://xxx.r2.cloudflarestorage.com/...`)
- Legacy `/uploads/` paths

Parameters:
- `url` (string): R2 URL or `/uploads/` path

Returns: `string|null` - R2 key or null

**`fileExistsInR2(key)`**

Checks if file exists in R2.

Parameters:
- `key` (string): File path in bucket

Returns: `boolean` - Exists status

## Database

Image URLs are stored as full R2 URLs in these columns:

**playlists**
- `image` (TEXT): Playlist cover image

**curators**
- `profile_image` (TEXT): Curator avatar

**tracks**
- `artwork_url` (TEXT): Track artwork
- `album_artwork_url` (TEXT): Album cover

**search_editorials**
- `image_url` (TEXT): Editorial image

### URL Format

**R2 URLs:**
```
https://your-r2-public-url.r2.dev/playlists/abc123_large.jpg
https://your-r2-public-url.r2.dev/curators/def456.jpg
```

**Legacy URLs (backward compatible):**
```
/uploads/playlists/abc123_large.png
/uploads/curators/def456.jpg
```

## Configuration

### Environment Variables

Required in `/etc/environment` (production) or `ecosystem.config.cjs` (development):

```bash
R2_ACCOUNT_ID="your-r2-account-id"
R2_ACCESS_KEY_ID="your-r2-access-key-id"
R2_SECRET_ACCESS_KEY="your-r2-secret-access-key"
R2_BUCKET_NAME="your-bucket-name"
R2_PUBLIC_URL="https://your-r2-public-url"
```

### Bucket Setup

Bucket name: `flowerpil-images`

Public access: Enabled via Cloudflare R2 dashboard

CORS configuration: Set via upload metadata
```javascript
Metadata: {
  'access-control-allow-origin': '*'
}
```

Cache control: `public, max-age=31536000, immutable`

### Directory Structure

R2 bucket paths:
```
/playlists/         - Playlist cover images
/curators/          - Curator profile images
/bio-pages/         - Bio page content and banner images
/search-editorials/ - Editorial images
/playlist-action-icons/ - Custom action icons
/tracks/            - Track artwork (manual uploads only, not used by scheduled imports)
```

### Detect R2 vs Local Path

From `server/utils/r2Storage.js` (lines 104-119):

```javascript
export function extractR2Key(url) {
  if (!url || typeof url !== 'string') return null;

  // If already an R2 URL, extract key
  if (url.includes('.r2.cloudflarestorage.com/')) {
    const parts = url.split('.r2.cloudflarestorage.com/');
    return parts[1] || null;
  }

  // If legacy /uploads/ path, convert to R2 key
  if (url.startsWith('/uploads/')) {
    return url.replace(/^\/uploads\//, '');
  }

  return null;
}
```

### Process and Upload Track Artwork

From `server/api/artwork.js` (lines 267-298):

```javascript
async function processAndSaveArtwork(buffer, filename) {
  const baseName = filename.replace(/\.[^/.]+$/, '');
  const extension = '.jpg';

  const sizes = [
    { suffix: '', width: 600, height: 600 },      // Default
    { suffix: '_lg', width: 800, height: 800 },   // Large
    { suffix: '_md', width: 300, height: 300 },   // Medium
    { suffix: '_sm', width: 150, height: 150 }    // Small
  ];

  // Process and upload different sizes to R2
  for (const sizeConfig of sizes) {
    const processedBuffer = await sharp(buffer)
      .resize(sizeConfig.width, sizeConfig.height, {
        fit: 'cover',
        position: 'center'
      })
      .jpeg({ quality: 85, progressive: true })
      .toBuffer();

    const r2Key = `tracks/${baseName}${sizeConfig.suffix}${extension}`;
    await uploadToR2(processedBuffer, r2Key, 'image/jpeg');
  }

  // Return R2 URL for the default size
  return `${process.env.R2_PUBLIC_URL}/tracks/${baseName}${extension}`;
}
```

## Deployment

### Production

1. Add environment variables to `/etc/environment`
2. Add R2 variables to `inheritEnv` array in `ecosystem.config.cjs`:
   ```javascript
   const sharedSecrets = inheritEnv([
     // ... other vars ...
     'R2_ACCOUNT_ID',
     'R2_ACCESS_KEY_ID',
     'R2_SECRET_ACCESS_KEY',
     'R2_BUCKET_NAME',
     'R2_PUBLIC_URL'
   ]);
   ```
3. Restart PM2 with environment loaded:
   ```bash
   set -a && source /etc/environment && set +a && pm2 restart ecosystem.config.cjs --env production
   ```
   Note: Use `set -a` to export all variables before sourcing
4. Verify R2 vars are loaded: `pm2 env 0 | grep R2`
5. New uploads automatically use R2
6. Legacy local file deletion continues to work

### Rollback

Database already contains R2 URLs. To rollback:

1. Revert code changes to use `fs.writeFileSync()`
2. Update environment to remove R2 variables
3. Restart application

Local files remain in `storage/uploads/` for fallback during migration period.
